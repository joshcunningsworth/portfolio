Grading Application

Grading Application

This program uses a lambda expression to define an operation on a set of values then it reads the values from a file and displays them to the user. It takes the student ID and grades from the file and calculates what the student's letter grade would be.

For example: if the student has an 80 - 89 in the course their letter grade would be an A, or if they have a 90 - 100 their letter grade would be an A+. This application would be useful for calculating student grades and averages. It's also practical for parsing through files and outputting the data. An example of the program reading the data from a file and outputing would look similar to the example below.

    1243395 94
    1269780 73.8

    Output from the program:
    1024576 51.37 D
    1243395 94.00 A+
    1269780 73.80 B

Garden Application

Garden Application

This program will allow a user to virtually plan a garden. They can enter the size of the garden and where they want crops/plants to be planted. It will accept a character symbol that represents the crop/plant in the garden.

For example: 'c' could symbolize carrots and the user could decide to enter 'w' as a new crop of wheat, assuming no other crops were using that character because each crop requires a unique symbol that cannot be shared. The user can add crops to their garden setup and view what has already been planted. The view of the garden is output in a text based grid with crops inside that looks similar to the grid below.

2D Dodge Game

2D Dodge Game

A basic 2D game where you play as an orange ball with the objective of dodging falling red squares for as long as possible. The game uses collision detection on the floor/stage, as well as the red squares, and orange ball. The games controls are input through a key listener, when the user presses the left or right arrow key the orange ball will move in the corresponding direction. Alternatively the user can use the 'A' key or 'D' key for left and right movement. When the ball is hit by a falling square the player loses a life and returns to the centre of the stage. When all three lives are lost the game is over.

It's a simple concept for a game but was something I worked on as a side project to practice Java. It was also good practice for collision detection and key inputs. Future plans for the game are to implement a difficulty system.

    SELECT department_id, MAX(hire_date) AS "Latest",

    FROM employees
